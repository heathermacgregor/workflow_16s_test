IMPLEMENTATION DETAILS
======================

Standardized Access Pattern:
```python
# OLD (non-standardized):
api_key = os.getenv("API_KEY")  # env-only, no config

# NEW (standardized):
from workflow_16s.utils.credential_utils import get_credential
api_key = get_credential(config, 'api_key')  # config→env→error
```

Config-First Fallback:
```python
# Tries config.credentials.api_key first
# Falls back to os.getenv("API_KEY") if config value is None
# Returns None or raises ValueError based on required flag
```

Email Fallback Logic:
```python
# get_email(config, 'ena_email') tries:
# 1. config.credentials.ena_email
# 2. config.credentials.email (fallback if ena_email not set)
# 3. os.getenv("ENA_EMAIL")
# 4. os.getenv("EMAIL")
# Raises ValueError if nothing found
```

SECURITY ANALYSIS
=================

Before Standardization:
- RISK: Credentials written to os.environ at initialization
- RISK: Credentials persisted in process environment
- RISK: Unencrypted in memory during entire process lifetime
- RISK: Exposure to environment introspection tools

After Standardization:
- SECURE: Credentials accessed on-demand from config
- SECURE: No implicit writes to os.environ
- SECURE: Shorter exposure window
- SECURE: Easier to audit credential access
- SECURE: Single source of truth (config)
